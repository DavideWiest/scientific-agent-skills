# Scientific Schematics

Adapted from K-Dense's upstream `scientific-schematics` skill as an optional
communication tool rather than a mandatory document requirement.

## Workflow

1. Decide whether a schematic materially clarifies the question, method,
   mechanism, or result. Do not add a figure merely to satisfy a quota.
2. Define the figure's message, audience, required labels, evidence-supported
   content, and any uncertainty that must remain visible.
3. Choose an available creation route suitable for the artifact: code-native
   diagram, plotting/vector tooling, or a configured image-generation tool.
4. Check scientific correctness, label legibility, accessibility, and whether
   the image implies unsupported causality or precision.
5. Store editable/source artifacts alongside rendered output when the figure
   is part of a durable project deliverable.

Never invent experimental structure or biological/mechanical detail for visual
appeal.
